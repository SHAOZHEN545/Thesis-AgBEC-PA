# 🎓 PhD 知识库 Git 同步工作流与配置指南

> **核心理念**：利用 Git 进行精确的版本控制，追踪思路演变，告别网盘同步冲突。Git 非常聪明，在没有文件变动时，同步操作会自动忽略，绝不会产生多余的“垃圾历史”。

---

## 🔄 跨设备切换工作流 (肌肉记忆)

为了保证家里、办公室等多设备之间的无缝流转，避免因为自动同步的时间差导致进度未上云，请务必养成以下习惯：

### 🎬 场景一：结束工作（下班/关电脑前）
**动作：手动触发一次全局同步。**
操作方法（任选其一，推荐方法 B）：
* **方法 A（侧边栏操作）：** 点击左侧边栏的 **“Source Control”（树状分支）图标**，在展开的面板顶部，点击 **“Sync”（首尾相接的双箭头）图标**。插件会自动完成 `Commit -> Pull -> Push` 的全套动作。
* **方法 B（快捷键，最快）：** 按下 `Ctrl + P`（Mac 为 `Cmd + P`）呼出命令面板，输入 `Git sync`，回车选择 `Obsidian Git: Commit and push all changes`。

*(注：界面右下角的 Git 状态栏主要用于静默查看当前的修改数量、分支名以及后台同步状态。)*

### 🚀 场景二：开始工作（刚到家/刚打开电脑）
**动作：无感等待。**
由于已开启“启动时拉取”，打开 Obsidian 后只需稍等几秒。插件会在后台自动执行 `Pull`，把云端的最新版本拉取到本地。当看到状态栏完成提示后，即可放心开始写论文。

## ⚙️ Obsidian Git 核心防暴涨配置备忘录

如果未来需要在新电脑上重新配置，请核对以下两处关键设置，这是保持 `.git` 文件夹轻巧、不臃肿的秘诀。

### 1. 必配 `.gitignore` (过滤网)
Vault 根目录下的 `.gitignore` 文件用于屏蔽极其高频且无意义的本地工作区状态变动，以及大体积附件，详细内容见该文件。

### 2. 插件核心参数 (Settings -> Obsidian Git)

- **[Automatic] 自动化备份兜底**
    
    - `Auto commit-and-sync interval (minutes)`: 设为 **30** 或 **60**。作为忘记手动同步时的兜底机制。
        
- **[Pull] 拉取设置**
    
    - `Pull on startup`: **开启**。这是实现多设备“无缝衔接”最重要的一步。
        
- **[Commit-and-sync] 同步逻辑**
    
    - `Push on commit-and-sync`: **开启**。
        
    - `Pull on commit-and-sync`: **开启**。
        
- **[Miscellaneous] 杂项与防打扰**
    
    - `Disable informative notifications`: **开启**。隐藏每半小时弹出的成功提示，保护沉浸式心流。
        
    - `Show status bar`: **开启**。保留右下角的低调状态显示。
